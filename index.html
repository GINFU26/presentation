The recommendation

Across both layers — keep Fluid as the default.

No external mechanism edge I found is large enough to outweigh the Microsoft integration still to be built.

Fluid wins on both layers
  • On mechanism, its structured state is native.
  • On platform, identity, storage, and compliance are already wired in.
External frameworks still matter — where Fluid should keep improving
First-class rich-text path Clearer Copilot edit patterns Easier onboarding for product teams

Why broker matters
1
Failure recovery. A broker log keeps ordering alive through a crash; brokerless shapes reload from storage.
2
Scale-out. A broker partitions documents across workers; brokerless pins each to one process.

Redpanda swaps in with no new code — 13× less average broker CPU, 33× less peak, 2.2× less memory, and one fewer container than Kafka + ZooKeeper.
